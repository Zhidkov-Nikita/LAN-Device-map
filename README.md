# LAN Device Map

Интерактивная 2D-карта офиса на Django с конструктором планировки для администратора.

Комнаты отображаются на Canvas (Fabric.js) как полупрозрачные прямоугольники с названиями.
Внешний контур здания статичен, внутренняя планировка полностью управляется из базы данных.

---

## Возможности

- **Режим просмотра** — все комнаты отображаются с цветовой индикацией, при наведении — подсветка, при клике — информация о комнате.
- **Режим редактирования** (staff only) — перетаскивание (drag) и изменение размера (resize) комнат на холсте. Изменения сохраняются в БД через AJAX.
- **API** — JSON-эндпоинт с данными комнат и эндпоинт для обновления координат.
- **Аутентификация** — кнопка «Войти» для анонимных пользователей, панель управления для администрации, кнопка «Выйти» для остальных.

## Быстрый запуск

### 1. Клонирование

```bash
git clone <repository-url>
cd LAN-Device-map
```

### 2. Виртуальное окружение

```bash
python -m venv venv
venv\Scripts\activate
```

### 3. Зависимости

```bash
pip install -r requirements.txt
```

### 4. База данных (PostgreSQL)

Скопируйте `.env.example` в `.env` и отредактируйте параметры подключения:

```
DB_NAME=landevicemap
DB_USER=postgres
DB_PASSWORD=postgres
DB_HOST=localhost
DB_PORT=5432
SECRET_KEY=django-insecure-your-secret-key-here
```

Убедитесь, что PostgreSQL запущен и база данных `landevicemap` существует.

### 5. Миграции

Создает таблицы в базе данных с базовыми значениями

```bash
python manage.py migrate
```

### 6. Создание администратора

Создайте администратора для теста админ панели и редактирования

```bash
python manage.py createsuperuser
```

### 7. Запуск

Локальный хост для просмотра локально

```bash
python manage.py runserver
```

Откройте `http://localhost:8000/` в браузере.

`http://localhost:8000/admin` админ панель.

## Запуск тестов

```bash
python manage.py test map
```

Будут выполнены:

- **RoomModelTest** — создание, значения по умолчанию, `__str__`, уникальность `code`, сортировка.
- **HomePageTest** — статус 200, наличие комнат и `rooms_json` в контексте.
- **SaveRoomsAPITest** — успешное сохранение админом, 403 для анонима и обычного пользователя, массовое обновление, обработка невалидного JSON.

## Режимы работы

### Просмотр

- На холсте отображается внешний контур здания и блоки комнат, загруженные из БД.
- При наведении — подсветка (повышение непрозрачности заливки).
- При клике — панель с названием и кодом комнаты.
- Перетаскивание и изменение размера отключены.

### Редактирование (staff only)

- Кнопка **«Редактировать»** переключает режим.
- Комнаты становятся жёлтыми, появляются угловые маркеры для изменения размера.
- Администратор может свободно двигать комнаты и менять их размер.
- При изменении размера название автоматически центрируется.
- Кнопка **«Сохранить»** отправляет POST-запрос на `/api/rooms/save/` с JSON-массивом `{id, x, y, width, height}` для всех комнат.
- После сохранения режим редактирования автоматически выключается.

## API

| Метод | Путь | Описание | Доступ |
|-------|------|----------|--------|
| GET | `/` | Главная страница с картой | Все |
| GET | `/api/rooms/` | JSON-список комнат | Все |
| POST | `/api/rooms/save/` | Сохранение координат комнат | Только администрация |

Ответ GET-запроса `/api/rooms/`:

```json
[
  {
    "id": 1,
    "name": "Архив",
    "code": "ARCH-01",
    "x": 750,
    "y": 31,
    "width": 220,
    "height": 289
  }
]
```
